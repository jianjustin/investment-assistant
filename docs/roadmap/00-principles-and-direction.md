# 00 — 方向判断与系统定位

> 本篇是整个 roadmap 的**地基论证**。后面所有阶段的优先级排序，都由这里的判断推导而来。先读这篇，再读阶段文档才不会觉得「为什么回测被排到后面」「为什么宏观被砍」。

---

## 1. 方向判断：对的部分 vs 危险的部分

### 1.1 方向对的地方

- **「数据地基 → 验证 → 决策/复盘」的排序原则正确**：先验证后决策，不在未验证的信号上叠加 LLM 叙事。
- **坚持人在环路、不自动下单**：诚实且正确。系统定位是降低决策成本与审计纪律，不是替你扣扳机。
- **隐含定位是「决策支持」而非「自动 alpha 工厂」**：对。下面会把这个定位讲透，因为它决定了工程重心。

### 1.2 必须先纠正的认知风险

第一版 roadmap 把「回测引擎」捧成核心分水岭。这是量化新手最容易掉进去的坑：

> **你的标的池只有几十只关注股 + 一个判别式打分。对单标的信号做组合回测，统计上几乎没有意义，而且是一台完美的过拟合机器。**

三个致命偏差，现状方向会直接踩中：

1. **样本量不足**：5–10 只股票上的几十个信号事件，无法支撑任何「胜率 / 盈亏比」结论。任何看起来漂亮的数字都是噪声，置信区间宽到没有决策价值。
2. **幸存者偏差**：watchlist 是「已经被你选出来、表现还行」的股票。用它回测等于已经偷看了答案——你测的不是策略，是你的选股后视镜。
3. **多重检验 / 数据窥探**：用回测结果去「反推权重」（第一版明确建议过，**现已撤销**）是在**保证过拟合**。你会调出一组历史完美、未来必然失效的参数。

补充一个区制风险：你的回测窗口大概率落在 2020–2024 这一段以涨为主的单一市场区制里，结论不可外推到震荡 / 熊市。

**结论**：回测要做，但要从「验证一个可交易策略」**降级**为「验证我关注的信号到底有没有边际信息」，并采用统计上更诚实的形式（见第 2 节）。

## 2. 把「回测」拆成三个严肃度层级

按 ROI 和统计诚实度排序。优先做 A、B，而不是一上来做 C。

| 层级 | 形式 | 回答的问题 | 统计可信度 | 工程量 | 建议 |
|------|------|-----------|-----------|--------|------|
| **A. 事件研究 (Event Study)** | 信号触发后 +1/+5/+20 日的前瞻收益分布 | 「这个信号后面平均会怎样？」 | **高**（可跨标的池化样本） | 小 | **先做** |
| **B. 横截面 IC / 命中率** | 评分与未来收益的秩相关、分档单调性 | 「分数高的真的比分数低的好吗？」 | 中高 | 中 | 第二步 |
| **C. 全组合回测** | 含仓位 / 成本 / 滑点的净值曲线 | 「整套策略历史净值如何？」 | **低**（小样本易过拟合） | 大 | 最后，且仅当 A/B 显示有边际才做 |

### 为什么 A 层最关键

事件研究把**分散的信号事件池化成足够样本**，绕开了单标的小样本问题。更重要的是，它直接接上你目标②里的**财报采集**：

> 「我池子里的股票 EPS beat 之后，平均 +5/+20 日跑赢大盘多少？」

这是教科书级的 **PEAD（Post-Earnings-Announcement Drift，财报后漂移）** 事件研究——有数十年学术文献背书、真实存在的市场异象。它比「对一个判别式打分做组合回测」靠谱十倍：样本可池化、有先验、结论可证伪。

A 层落地细节见 [01-phase0](01-phase0-data-quality-and-event-study.md)。

## 3. 真正的投研系统该有、但旧 roadmap 漏掉的支柱

按重要性排序：

### 3.1 组合 / 敞口视角（最大的结构性缺失）

整个系统是「单股信号」导向的，但投研的核心问题是 **「我现在整体暴露在什么风险上」**：当前持仓、行业 / 因子集中度、相关性、Beta、净 / 总敞口、回撤、现金比例。

关键洞察：**这个数据就藏在你目标③的交易日志里——日志 = 真实持仓**。把 `trade_journal` 升级成「可推导组合状态」而不仅是「流水」，价值翻倍。详见 [05-cross-cutting](05-cross-cutting.md)。

### 3.2 计划→执行→结果→复盘的闭环（你三个目标真正的连接点）

这是别人都不建、但杀伤力最大的功能。如果 `execution_plans`（目标②的产物）和 `trade_journal`（目标③的产物）**共享主键**，你就能量化两件事：

- **计划遵守度**：实际成交价 / 仓位 vs 自己定的进场 / 止损 / 目标，偏离多少？
- **「听自己话」是否赚钱**：遵守计划的交易 vs 临时起意的交易，期望值差多少？

目标②（计划）和目标③（复盘）本质是**同一个闭环的两端**，不该当成两条独立主线。这是 Phase 2 的核心，见 [03](03-phase2-plan-execution-loop.md)。

### 3.3 市场状态：广度指标 > 重仓 FRED 宏观（自我修正）

第一版把 FRED 宏观排得很靠前。**纠正**：

- 宏观择时的历史战绩很差；CPI / 就业是**滞后数据**，对择时基本是噪声。
- 对一个股票交易者，**市场广度**（% 个股站上 200 日线、新高新低、涨跌家数线 A/D Line）是更便宜、更灵敏、更有效的状态信号——而且用你**已经在拉的价格数据**就能算，几乎零额外数据成本。
- 宏观里真正有 risk-off 价值的只有 **HY 信用利差**（`BAMLH0A0HYM2`），其次是收益率曲线（2s10s）。

**重排后的市场状态因子优先级**：

```
市场广度  >  HY 信用利差  >  收益率曲线(2s10s)  >>  CPI / 就业 / Fed Funds
```

别为采集一堆滞后宏观指标做大工程。`config.py` 里 `MacroConfig` 已列了 6 个 FRED series，**精简到 HY 利差 + 2s10s 两个即可**，其余推迟。详见 [02](02-phase1-data-foundation.md)。

### 3.4 复盘要做「跨笔模式识别」，不是「单笔叙事」

DeepSeek 在复盘里的**正确用法**不是「给这笔交易写段点评」（价值低），而是在几十上百条日志上**找行为模式**：

- 是否在高 VIX 时过度交易？
- 是否赢小利就跑、亏了死扛（截断利润 / 放任亏损 = disposition effect）？
- 是否总在同一类信号 / 同一行业上亏钱？

并算出你实盘的真实**期望值、胜率、盈亏比、Sharpe**，再和「无脑持有 SPY」「策略理论最优」做**三方对比**。这才是复盘的火力点。见 [04](04-phase3-review-and-regime.md)。

### 3.5 区制条件化（连接目标①和②的隐藏价值）

给每个信号 / 每笔交易打上「当时的市场状态」标签，所有统计按区制分层。

> 「RS 策略在 green 有效、在 red 失效」——这种结论既可执行，又把你的市场状态判断（目标①）和个股策略（目标②）真正接在一起。

现状已有便利：`market/service.py` 的 `compute_market_signal_for_date(...)` 已支持按历史日期计算市场状态，区制标签可直接回填。

### 3.6 LLM 校准追踪（让 DeepSeek 可问责）

只要 Hermes / 系统输出带概率的判断（「70% 守住 21EMA」），就**记录下来**，事后算 **Brier score / 校准曲线**。否则 LLM 只会产出自信的废话，你无法知道它到底准不准。这是 LLM 驱动系统少有人做、却决定可信度的一环。见 [04](04-phase3-review-and-regime.md)。

## 4. 系统定位（一句话锁死工程重心）

> **护城河不在「又一个量化回测引擎」（那个赛道样本量赢不了），而在「把我的真实交易决策、当时的市场状态、事后结果，闭环起来做归因」——这是只有你自己的数据才能做、且大多数工具都没有的东西。**

由此推出三条工程纪律：

1. **闭环优先于回测**：`计划-执行-复盘` 共享主键的闭环（Phase 2）比全组合回测（被推迟的 C 层）优先。
2. **证伪优先于建设**：Phase 0 事件研究先回答「信号有没有用」，没用就不在上面叠加。
3. **诚实优先于漂亮**：所有统计带样本量与置信区间；拒绝用历史反推权重；区制分层报告，不报单一区制的漂亮总数。

## 5. 修正后的优先级（推导结果）

```
Phase 0  事件研究 + 数据质量地基       ← 最小成本验证「信号有没有边际」
Phase 1  OHLCV落库 + 市场广度 + EPS surprise结构化   ← 只补被证明有用的数据
Phase 2  execution_plans + trade_journal 共享主键闭环  ★系统价值核心
Phase 3  跨笔行为模式 + 三方对比 + 区制条件化 + LLM校准  ← 复盘火力
横切:    组合敞口 / 可视化 / 文档 / 安全 / CI
```
