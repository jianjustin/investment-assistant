# 06 — Backlog：高性价比小项

> 这些都不大，但对投研日常很实用。不绑定阶段，**随时可插**到任意阶段的空隙里。按性价比 / 依赖排序。

---

## 即用型（依赖已有数据，可尽早做）

### B1 — 前瞻财报日历
- **做什么**：知道未来一周池子里**谁要发财报**（现状是事后下载，应改为事前预警）。
- **数据**：yfinance `earnings_dates` 未来窗口；接现有 scheduler + Discord 通知。
- **价值**：财报是事件驱动交易的核心节点，事前预警 >> 事后下载。与 Phase 0 的 PEAD 事件研究天然互补（一个是事后统计漂移，一个是事前提醒布局）。

### B2 — 相对强弱全池排名
- **做什么**：不只 vs SPY，而是**池内百分位排名** + 行业分类，看轮动。
- **数据**：`tickers/trend.py` 的 RS + `ohlcv_bars`。
- **价值**：「谁在领涨、资金往哪轮」比单看一只股的 RS 有用。

### B3 — 「昨日至今变了什么」diff 视图
- **做什么**：投研每天第一眼想看的是**变化**，不是全量快照——评分变动、新触发信号、新进/退出关注、市场状态切换。
- **数据**：对比 `ticker_signal_snapshots` / `strategy_scores` 相邻日。
- **价值**：把「读一屏数字」变成「看几条变化」，每日扫描成本大降。

---

## 需少量计算（OHLCV 落库后）

### B4 — 波动率分位 / ATR 分位
- **做什么**：每只股票当前波动处于**历史什么水位**（如 ATR 在过去一年的百分位）。
- **价值**：是仓位 sizing 的直接输入（高波动小仓），喂给 Phase 2 的 `plan_builder`。

### B5 — 相关性矩阵
- **做什么**：watchlist 两两相关性，暴露**集中度风险**。
- **价值**：「我以为分散，其实持仓全是半导体同涨同跌」——与 C1 组合敞口呼应。半导体成长股池（CRDO/MU/NVDA）尤其需要这个。

### B6 — 成交量异常检测
- **做什么**：相对 N 日均量的放量倍数排名。
- **价值**：量是价的先行，异常放量常先于事件。

---

## 数据质量类（地基，易被忽略）

### B7 — 复权 / 公司行为校验（部分并入 Phase 0 Task 0.1）
- **做什么**：split / 分红调整正确性、退市 / 改名 / 代码变更处理。
- **价值**：yfinance 在这块不可靠，**回测与事件研究能不能信的隐形地基**。Phase 0 已纳入基础校验，B7 是其延伸（如自动比对多源、维护公司行为表）。

### B8 — 数据缺口监控
- **做什么**：`ohlcv_bars` 缺 bar、`fundamentals` 缺季度的自动告警。
- **价值**：静默的数据缺口会让统计悄悄失真。

## LLM / Hermes 增强（Phase 3 之后）

### B9 — 多步研究链路
- **做什么**：`agents.py` 已定义但未执行的角色（filing_digest / watchlist_research / 反方挑战）真正串成多步编排：宏观 → 个股 → 财报摘要 → 反方挑战。
- **价值**：从单次 LLM 调用升级为有结构的研究流程。依赖 Phase 1 结构化财报数据就绪。

### B10 — 邮件推送
- **做什么**：`config.py` 已有 `NotifyConfig.email_enabled`，但无 `notify/email.py` 实现。补 SMTP 客户端 + HTML 模板。
- **价值**：Discord 之外的第二推送渠道；复盘报告 / 财报预警适合邮件。优先级低于核心闭环。

## 优先级速查

```
最高性价比、可立刻做:   B1 财报日历 · B3 diff视图 · C3 文档(见05)
OHLCV落库后顺手:        B4 波动率分位 · B5 相关性 · B2 RS排名
地基不性感但重要:       B7 复权校验 · B8 缺口监控
锦上添花、靠后:          B6 放量 · B9 研究链路 · B10 邮件
```

> 这些项不应抢占核心闭环（Phase 2）的工程时间，但适合作为「等数据任务跑、等回测结果」时的填充工作，或你某天想做点轻量改进时的候选。
